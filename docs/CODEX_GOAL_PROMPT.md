请在当前 `Life Project` 项目中完成下一轮体验与数据修正。开始前先阅读：

- `docs/PROJECT.md`
- `docs/NEXT_STEP_PLAN.md`
- `docs/sql/01_tasks_schema.sql`
- `docs/sql/02_habits_import_export_schema.sql`
- `resource/Loop Habits CSV 2026-07-05.zip`

当前项目已经有 `/today`、`/habits`、`/heatmap`、`/import`、`/export`，以及 Supabase Auth、任务表、习惯模板、每日记录、导入作业和导出功能。请在现有结构上修正，不要重写整个项目。

## 目标

把当前功能从“基础可用”推进到“适合长期日常使用”，重点修正中文、本地化、移动端、导入准确性、完成度、分数、日期和单习惯详情。

## 必须完成

1. 中文化
- 所有按钮、导航、表头、状态、提示、空状态、错误文案都改为中文。
- 不要保留 `Today`、`Habits`、`Heatmap`、`Import`、`Export` 这类英文按钮。

2. PC 和手机端适配
- `/today`、`/habits`、`/heatmap`、`/import`、`/export`、登录页、注册页都必须适配 PC 和手机。
- 手机端不能出现按钮挤压、文字溢出、表格不可读、主要操作无法完成的问题。

3. 修复导入准确性
- 导入 `resource/Loop Habits CSV 2026-07-05.zip` 后，`睡觉`、`起床` 这类 `YES_NO` 习惯只能得到 `0` 或 `1`，不能出现小数。
- `Checkmarks.csv` 优先作为二值完成状态来源。
- `Scores.csv` 不能覆盖 `YES_NO` 习惯的二值语义。
- 没有数据的日期自动视为 `0`。
- 导入预览和导入结果中应能看出原始值、归一化值、完成状态和完成度。

4. 自动完成判断
- 每个习惯每天自动判断是否完成。
- `AT_LEAST`：实际值大于或等于目标值为完成。
- `AT_MOST`：实际值小于或等于目标值为完成。
- `YES_NO`：值为 `1` 或明确完成为完成。
- 无数据按 `0` 计算。

5. 每日完成度
- 每个习惯每天有 `completion_ratio`，范围 `0` 到 `1`。
- 完全达成为 `1`，完全没有完成为 `0`。
- `AT_LEAST`：`min(actual / target, 1)`。
- `AT_MOST`：小于或等于目标值为 `1`，达到 `2 * target` 为 `0`，中间线性下降。
- `YES_NO`：完成为 `1`，未完成或无数据为 `0`。
- 数值最多保留三位小数。
- 支持三位小数缩放规则，例如导入或展示中的 `7000` 应视为 `7`。

6. 分数表
- 每个习惯增加分数表或等价数据结构。
- 每个习惯每天有一个分数，范围 `0` 到 `100`。
- 分数受每日完成度影响。
- 越近的日期权重越高，越久远的日期影响越弱。
- 分数可重新计算，导入修正后能重新生成历史分数。

7. 日期与北京时间
- 所有业务日期统一按北京时间 `Asia/Shanghai`。
- 今日、日期切换、导入日期、导出日期、热力图范围都必须使用北京时间。
- 封装统一日期工具，避免页面中直接散落使用 `new Date()` 来决定业务日期。

8. 日历型日期修改
- `/today` 和 `/habits` 至少提供更方便的日历型日期选择。
- 支持快速回到今天、切换昨天/明天、选择具体日期。
- 手机端也必须易用。

9. 习惯删除
- 习惯可以删除。
- 删除前需要确认。
- 删除需要考虑历史记录、分数记录和导入映射。
- 如果为了安全选择软删除，请在 UI 上让用户能理解“归档”和“删除”的区别。

10. 单个习惯详情页
- 增加单习惯详情页，例如 `/habits/[id]`。
- 页面应包含习惯基础信息、单习惯热力图、完成度趋势、分数趋势，以及编辑/归档/删除入口。

11. 热力图和导出联动
- 热力图应使用新的完成度和分数逻辑。
- 导出数据应包含实际值、归一化值、完成状态、完成度和分数。
- 导出的数据应尽量能再次导入。

## 建议执行顺序

1. 先修正文案和响应式基础布局。
2. 再补统一的北京时间日期工具。
3. 再设计并实现归一化值、完成度、分数的数据结构和计算函数。
4. 再修复 `Loop Habits` 导入逻辑，尤其是 `YES_NO` 和 `Scores.csv` 覆盖问题。
5. 再改 `/habits`、`/today` 的日期选择和习惯删除。
6. 再新增 `/habits/[id]` 详情页。
7. 最后更新热力图、导出和验证流程。

- 导入 `resource/Loop Habits CSV 2026-07-05.zip` 后，`睡觉`、`起床` 只有 `0` 或 `1`。
- 每个习惯每天都有实际值、归一化值、完成状态、完成度和分数。
- 无数据日期按 `0`。
- 日期按北京时间。
- 所有主要页面 PC 和手机端可用。
- 每个习惯都有独立详情页。
- 导出结果包含新字段，并能用于备份和再次导入。
