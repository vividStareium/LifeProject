# Life Project

## 1. 产品目标

打造一个支持跨设备同步的日程与每日任务 Web App。

- 用户在手机或电脑打开同一网址，用同一账号登录
- 可随时查看与编辑当日及其他日期任务
- 所有改动实时同步到云端，保证多端一致性

## 2. V1 范围

V1（最小可交付版本）仅覆盖以下能力：

1. 用户身份体系
   - 注册、登录、退出
2. 今日任务页
   - 页面路径：`/today`
   - 默认展示当前日期任务
3. 日期切换
   - 支持查看历史/未来日期任务
4. 任务管理
   - 新增任务：标题、描述、日期、开始时间、结束时间、优先级、分类
   - 任务列表按时间排序（无时间任务排在后面）
   - 勾选完成 / 取消完成
   - 软删除任务（保留历史）
5. 简单统计
   - 今日总任务数
   - 今日已完成数
   - 今日未完成数
6. 自适应布局
   - 在手机与桌面浏览器均可正常使用

> V1 不包含：共享协作、复杂权限模型、提醒推送、日历外部集成、标签体系扩展与高级统计报表。

## 3. 技术栈

- 前端：Next.js App Router + TypeScript
- UI：Tailwind CSS
- 认证：Supabase Auth
- 数据库：Supabase Postgres
- 部署：Vercel
- 运行平台：Web（首要兼容移动端与桌面端）

## 4. 数据库设计

### 4.1 表：`tasks`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `uuid` | 主键，任务唯一 ID |
| `user_id` | `uuid` | 归属用户（关联 `auth.users(id)`） |
| `title` | `text` | 任务标题，不能为空 |
| `description` | `text` | 任务描述 |
| `task_date` | `date` | 任务所属日期 |
| `start_time` | `time` | 开始时间，可空 |
| `end_time` | `time` | 结束时间，可空 |
| `status` | `text` | 状态：`todo` / `done` / `cancelled` |
| `priority` | `text` | 优先级：`low` / `medium` / `high` |
| `category` | `text` | 分类（文本标签） |
| `created_at` | `timestamptz` | 创建时间 |
| `updated_at` | `timestamptz` | 更新时间 |
| `deleted_at` | `timestamptz` | 软删除时间戳，非空表示已删除 |

### 4.2 设计约束（建议）

- `title` 需非空；`status`、`priority` 建议使用约束/枚举值校验
- `user_id` 加索引，用于按用户查询
- `(user_id, task_date)` 复合索引，提升按日期列表查询性能
- `deleted_at IS NULL` 作为默认筛选条件，支持软删除语义
- 启用 RLS，按 `user_id` 与登录用户绑定；仅允许用户操作自己的任务

### 4.3 SQL 设计注意事项（建表示意）

- 字段命名保持与前端一致，避免中间层映射成本
- 时间字段保存为 UTC（`timestamptz`）并在显示层格式化成本地时间
- 如需后续扩展，优先追加新字段而非重构现有列类型

## 5. 开发原则

- 先满足需求再优化，不预置未在 V1 说明中的功能
- 每次改动先说明计划，再执行实现
- 每次提交或变更后明确列出改动文件清单
- 遇到命令执行需求，先向用户说明具体命令
- 若需数据库操作，提供可直接执行的完整 SQL
- 遵循最小闭环：先让 V1 可运行，再逐步扩展
